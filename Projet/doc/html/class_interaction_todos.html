Detailed Description

Container for InteractionTodo objects.

A container class for InteractionTodos. Represents the idea that a Relational Database only has atomic values
It is not a list of pointers because an InteractionTodo has no reason to exist out of this list. Therefore every time there is the need to use an element of this list, a pointer towards it has to be created.

Example

Here is an example of the itList attribute of an InteractionTodos object with the following interaction :
Interaction * i = new Interaction("Please Valentin we really need to finish this school project. @TODO Brew some coffee @TODO Rush the remaining work")

1st element of itList (InteractionTodo type)
attribute Iattribute T
inew Todo("Brew some coffee")
2nd element of itList (InteractionTodo type)
attribute Iattribute T
inew Todo("Rush the remaining work")



Relational Database point of view

The whole point of this hard work is to be able to easily work with the database. The previous example easily translates into a Relational Database table :

void InteractionTodos::addInteraction ( Interaction *  interaction)

Method adding to the container the list of InteractionTodo objects from a given Interaction.
As in, the set of links between the given Interaction and all its todos.

Example

Interaction * i new Interaction("The quick brown fox @TODO jumps over @TODO the lazy dog");
addInteraction(i);

Should add two elements to the list :
new InteractionTodo(i,new Todo("jumps over")) and new InteractionTodo(i,new Todo("the lazy dog"))

interactionThe interaction to retrieve the Todos from.
