Detailed Description

An InteractionTodo objects serves the purpose of linking an Interaction object with a Todo object through their pointers.

An InteractionTodo object has for sole purpose to link a pointer on Todo and a pointer on Interaction.
It conceptually means "Interaction i CONTAINS Todo t"
It is similar to the way a Relational Database works : Attributes are Atomic since we associate a single interaction with a single todo.
Somewhere in the program will be an InteractionTodos object that acts as a list of InteractionTodo (with a few useful methods to manipulate them) to store all the possible links between existing Interaction objects and Todo objects.

Example

Here is an example of an InteractionTodo object with both its attributes :

attribute Iattribute T
new Interaction("Huge meeting @TODO call the boss")new Todo("call the boss")
