\doxysubsection{Detailed Description}
An \mbox{\hyperlink{class_interaction_todo}{Interaction\+Todo}} objects serves the purpose of linking an \mbox{\hyperlink{class_interaction}{Interaction}} object with a \mbox{\hyperlink{class_todo}{Todo}} object through their pointers. 

An \mbox{\hyperlink{class_interaction_todo}{Interaction\+Todo}} object has for sole purpose to link a pointer on \mbox{\hyperlink{class_todo}{Todo}} and a pointer on \mbox{\hyperlink{class_interaction}{Interaction}}.~\newline
 It {\bfseries{conceptually means \char`\"{}\+Interaction i CONTAINS Todo t\char`\"{}}}~\newline
 It is similar to the way a Relational Database works \+: Attributes are {\bfseries{Atomic}} since we associate a single interaction with a single todo.~\newline
 Somewhere in the program will be an \mbox{\hyperlink{class_interaction_todos}{Interaction\+Todos}} object that acts as a list of \mbox{\hyperlink{class_interaction_todo}{Interaction\+Todo}} (with a few useful methods to manipulate them) to store all the possible links between existing \mbox{\hyperlink{class_interaction}{Interaction}} objects and \mbox{\hyperlink{class_todo}{Todo}} objects.\hypertarget{class_todo_ex}{}\doxysubsubsection{Example}\label{class_todo_ex}
