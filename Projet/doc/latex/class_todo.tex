\doxysubsection{Detailed Description}
Represents a task from an interaction for a contact. Linked to \mbox{\hyperlink{class_interaction}{Interaction}} objects using an \mbox{\hyperlink{class_interaction_todo}{Interaction\+Todo}} object. 

A todo object conceptually represent a SINGLE task to do, ponctually, in an interaction that can contain as many todos as it needs.~\newline
 Todos are defined using {\bfseries{@\+TODO}} tags in an interaction.~\newline
 Todos dates are defined using {\bfseries{@\+DATE}} tags in a todo.~\newline
 If no date is defined, then it is implied to be the current date!~\newline
 When printing a todo object, we do not need the tags since they are only needed to retrieve them from interaction contents.\hypertarget{class_todo_ex}{}\doxysubsubsection{Example}\label{class_todo_ex}
Consider the following interaction object \+:~\newline
~\newline
 
\begin{DoxyDescription}
\item[\mbox{\hyperlink{class_interaction}{Interaction}}\{  ]content = \char`\"{}\+Please Valentin we really need to finish this school project. @\+TODO Brew some coffee @\+TODO Rush the remaining work tomorrow (the @\+DATE 26/10/2021)\char`\"{}~\newline
 date = \mbox{\hyperlink{class_date}{Date}}(23/10/2021)~\newline
\}
\end{DoxyDescription}~\newline
~\newline


The todo objects we want to build here are \+:~\newline
~\newline
 
\begin{DoxyDescription}
\item[\mbox{\hyperlink{class_todo}{Todo}}\{  ]content = \char`\"{}\+Brew some coffee\char`\"{}~\newline
 date = \mbox{\hyperlink{class_date}{Date}}(23/10/2021)~\newline
\}
\end{DoxyDescription}~\newline
 
\begin{DoxyDescription}
\item[\mbox{\hyperlink{class_todo}{Todo}}\{  ]content = \char`\"{} Rush the remaining work tomorrow (the 26/10/2021)\char`\"{}~\newline
 date = \mbox{\hyperlink{class_date}{Date}}(26/10/2021)~\newline
\}
\end{DoxyDescription}~\newline
~\newline
\hypertarget{class_todo_err}{}\doxysubsubsection{Errors}\label{class_todo_err}
There is only one error to manage here \+: Imputing a wrong date (incorrect format or whatever) -\/ Or imputing two or more dates.~\newline
 In both case, if there is an error, then the @\+DATE(S) will simply remain, not be treated, and the todo will be created at the current date
